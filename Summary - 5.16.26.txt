We built a sophisticated baseball betting model from scratch. The core model pulls data from MLB Stats API, Baseball 
Savant, and The Odds API, calculates expected runs using a multi-metric lambda (wOBA, xwOBA, FIP, xFIP, K%, BB%, 
rolling runs, park factors), runs 5,000 Monte Carlo simulations per game, and identifies edges on both moneylines 
and over/unders against DraftKings and FanDuel lines.

The model runs automatically three times daily via Windows Task Scheduler, logs all predictions to Google Sheets 
with run-by-run tracking, and produces clean terminal output showing edges, projected runs, and game statuses.

We built a backtesting framework using 11,040 historical games from 2021-2025 with FanDuel and DraftKings closing 
lines. Early results show the model finds profitable edges when filtering to favorites with 3-8% projected edge 
(+3.2% ROI historically). We identified a data leak where season-level stats included future game data — we re-pulled
detailed per-game logs to fix this, and the random parameter search is ready to run once the fix is implemented.

==========================
BACKTEST.PY SUMMARY - 5/17
==========================
Tests 100 random combinations of model parameters against 4 seasons of real historical games (2021-2024) to find
which settings produce the best betting ROI. 

For each trial it:
- Randomly picks values for things like edge thresholds, rolling average weights, FIP/wOBA weights, rating caps
- Loops through every game in the odds dataset for all 4 seasons
- Computes each team's stats using only data available before that game (no future leakage)
- Runs the Poisson simulation to get win probabilities
- Checks if there's an edge vs. the FanDuel line, places a simulated $100 bet if so
- Tracks win/loss and calculates ROI across all bets for that parameter set

What's saved:
- historical_data/search_YYYY-MM-DD_seed{N}_100trials.csv — one file per run, each row is one trial with all its 
params + ROI/win rate/n_bets/profit
- historical_data/random_search_checkpoint.json — temporary resume file saved every 10 trials, deleted when run 
completes

What's printed:
- Progress every 10 trials with elapsed time and ETA
- Total runtime when finished
- Top 10 parameter combinations by ROI for that run evaluate_runs() summary across all runs: per-run ROI summary, 
top 10 results combined, which param values appear most in top-25% trials, and correlation of each weight with ROI
