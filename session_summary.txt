STATCAST (Statcast/ folder):
- Files: batters_YYYY.csv and pitchers_YYYY.csv for 2021-2025
- Structure: one row per team per game. Key columns:
  player_name (team abbrev), game_date, babip, xwoba,
  barrels_per_pa_percent, pa
- Date format differs: batters use YYYY-MM-DD, pitchers use M/D/YYYY.
  Handled via pd.to_datetime() normalization.
- 7 abbreviation mismatches: AZ→ARI, CWS→CHW, KC→KCR, SD→SDP,
  SF→SFG, TB→TBR, WSH→WSN (stored in STATCAST_TEAM_MAP dict)
- Preprocessing: load_statcast_data() loads CSVs, normalizes dates/abbrevs,
  caches to historical_data/statcast_cache.json
- Runtime: precompute_statcast() sweeps each team's games once using sorted
  date sweep — O(games + target_dates) per team. Computes PA-weighted
  cumulative xwOBA, BABIP, barrel% up to each target date. Also computes
  league averages per date. Called once per season per trial.

RELIEF PITCHER STATCAST (rp/ folder):
- 13 CSV files covering all 6 divisions, 2021-2025, 80,312 rows total.
  No truncation (checked — no file hits exactly 10,000 rows).
  No duplicates (verified by player_id + game_date + game_pk).
- File naming varies (al_east_2021, nl_west_2021_2022, etc.) — doesn't matter,
  loader uses glob('rp/*.csv') to grab all files.
- Structure: player-level (one row per pitcher per game). Key columns:
  player_id (col B), game_date (col D), game_pk (col E),
  total_pitches (col F), babip (col J), woba (col L), xwoba (col M),
  k_percent (col AS), bb_percent (col AU),
  barrels_per_pa_percent (col BD), pa
- NO team column — player_id mapped to team via MLB Stats API.
- build_player_map.py: pulls 'fullSeason' rosters for all 30 teams for
  2021-2025 via statsapi.get('team_roster'). Stores as
  historical_data/player_team_map.json keyed by {season: {player_id: fg_abbrev}}.
  99.8% match rate (only Ross Stripling unmatched — traded mid-season).
- load_bullpen_data(): loads all rp/ CSVs, maps player_id to team, aggregates
  to team-game level (sum pitches, PA-weighted quality stats), sorts by date,
  caches to historical_data/bullpen_cache.json.
- precompute_bullpen(): sweep per team per season, computes:
  * pitches_7d: sum of pitches in last 7 calendar days (fatigue proxy)
  * xwoba, k_pct, bb_pct: cumulative PA-weighted season-to-date quality
  * League averages per date
- RESULT: bullpen data added but ultimately removed from param space after
  analysis showed w_bullpen_fat at -0.2456 correlation (actively harmful)
  and w_bullpen_qual at +0.007 (neutral/noise).

BULLPEN QUALITY COMPOSITE NOTE:
- Originally averaged xwOBA, K%, BB% equally for quality rating.
- Issue identified: xwOBA ratios cluster tightly (0.90-1.10) while K%/BB%
  ratios swing wider (0.70-1.40). Equal average lets wider-swinging stats
  dominate by variance. Since xwOBA already subsumes K%/BB% effects,
  equal weighting also double-counts that information.
- Decision: use xwOBA alone for quality (cleanest single metric, avoids
  scale mismatch). bp_qual_r = lg_xwoba / team_xwoba for pitching.

PROBLEM:
For every game in the backtest, get_rolling_averages_for_date() was called:
- Twice for home/away teams
- 30 times to compute all_team_rollings for league averages
- get_league_hr_fb_rate() scanned all 30 teams' pitching logs
Total: ~32 full log scans per game × avg 80 prior games × 9,600 games
× 100 trials = ~2.4 billion operations. ~102 minutes per 100 trials.

SOLUTION: precompute_all_rolling()
Called ONCE at module load before any trials run. For each season:
1. Per-team sweep: sorts hitting/pitching logs by date, builds prefix sum
   arrays for O(1) rolling window computation, sweeps through target dates
   in order accumulating cumulative wOBA/FIP/K%/BB% stats. O(games + dates).
2. League HR/FB rate: collects all pitching events across all teams, sweeps
   sorted by date, accumulates cumulative HR and FB counts. O(all_games).
3. Adds xFIP to each team entry using lg_hr_fb_by_date.
4. Computes league averages per date (woba, fip, xfip, k_pct, bb_pct, avg_runs).
5. Stores in two global dicts:
   rolling_team_cache[(season_str, team, date)] -> stats dict
   rolling_lg_cache[(season_str, date)] -> league averages dict

In run_backtest_with_params(), all per-game calls replaced with O(1) dict lookups.
Removed: get_rolling_averages_for_date(), get_league_hr_fb_rate(),
         all_team_rollings loop, calc_xfip() nested function.

RESULT: 8.3x speedup.
- 10 trials: 617s → 74s
- 100 trials: ~102 min → ~12 min
- 5 runs: ~8-9 hours → ~60 min
- 10 runs: ~17 hours → ~2 hours

CRITICAL FINDING — SAMPLE SIZE / OVERFITTING:
At n=90 bets, 95% confidence interval on win rate spans 21 percentage points.
The top small-sample trials (ROI 17-21%) show win rates of 64-67% — statistically
indistinguishable from luck at that sample size.

Key comparison:
- n >= 0:   top ROI +21.8%, favorites_only=True, win rate 67%, n=100 bets
- n >= 300: top ROI +14.7%, favorites_only=FALSE, win rate 50%, n=375 bets
- n >= 500: top ROI +11.3%, favorites_only=FALSE, win rate 50%, n=863 bets

favorites_only=True was a small-sample artifact. With narrow edge window
(0.07-0.08) + only favorites, you get ~90-135 bets over 4 seasons — too
few to distinguish skill from variance. At n>=300, all top-10 trials flip
to favorites_only=False.

evaluate_runs() now shows two top-10 tables: all trials + n>=300 filtered.
