1. BUG FIXES — model.py
------------------------
- Dead code in calculate_matchup(): get_total_line(game) was referencing the
  outer loop's global "game" variable silently and discarding the result.
  home_line/away_line were computed but never returned. Both removed.
- Three module-level debug lines fetching Yankees pitching data on every run
  were removed (avg_pitch_7, avg_pitch_15 — never used anywhere).
- UnicodeEncodeError: Windows terminal (cp1252) couldn't print emoji characters.
  Fixed by adding sys.stdout.reconfigure(encoding='utf-8') at the top.
- TypeError in log_predictions(): integer scores being written into a str-typed
  DataFrame column when updating existing rows. Fixed by wrapping in str().
- Google Sheets 429 rate limit: was calling sheet.append_row() once per row
  (~62 API calls). Replaced with a single sheet.update([headers] + rows) call.

  in the dict returned by get_rolling_averages_for_date(). Fixed keys to

  'Season' (capitalized). Fixed.

4. BACKTEST INFRASTRUCTURE
---------------------------
Added to backtest.py:
- Sleep prevention: SetThreadExecutionState keeps computer awake during runs
- Checkpointing: saves progress every 10 trials to JSON, resumes if interrupted
- Per-run file naming: search_YYYY-MM-DD_seed{N}_100trials.csv (each run
  preserved separately, never overwritten)
- Timer: prints elapsed time and ETA every 10 trials, total time at end
- Auto seed generation: new random seed each run so param space is explored
  differently each time. Seed printed so any run can be reproduced.
- N_RUNS loop: runs consecutive 100-trial searches automatically
- evaluate_runs(): pools all search CSVs, shows per-run summary, top 10 results
  across all runs, parameter breakdown (top 25% of trials), weight correlations
  with ROI. Saves combined CSV.

7. PERFORMANCE OPTIMIZATION
-----------------------------
Problem: For every game in the backtest, get_rolling_averages_for_date() was
called 32 times (home + away + 30 teams for league averages). Each call scanned
all prior games. With ~9,600 games × 100 trials × 32 calls × avg 80 prior
games = 2.4 billion operations.

Solution: precompute_all_rolling() — sweeps each team's game log ONCE per
season using prefix sums for O(1) rolling window computation. All stats for all
(team, date) combinations stored in a lookup dict before any trials run.

Result: 8.3x speedup. 10 trials went from 617s to 74s. 100 trials now takes
~12 minutes instead of ~102 minutes. 5 runs = ~1 hour instead of ~8-9 hours.
